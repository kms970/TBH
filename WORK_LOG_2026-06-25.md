# 2026-06-25 작업 기록

## 완료한 작업

- QHD/1.5x 환경에서 큐브 모드 전환이 잘못 클릭되던 문제를 수정했다.
- 보상상자 탐지 관련 기존 패치를 포함해 `v0.1.4` 릴리즈를 만들고 GitHub에 업로드했다.
  - https://github.com/kms970/TBH/releases/tag/v0.1.4
- 이후 "상자가 없을 때 다른 곳을 클릭한다"는 제보를 받고, 형태 기반 보상상자 후보가 영웅/큐브 UI 조각을 상자로 오인하는 문제를 확인했다.
- 첫 번째 대응으로 `v0.1.5`를 배포했다.
  - https://github.com/kms970/TBH/releases/tag/v0.1.5
  - 내용: 첫 보상상자 탐색에서 파란 상자와 갈색 상자가 한 쌍으로 보일 때만 클릭하도록 후보 필터를 강화했다.

## 새로 확인된 문제

`v0.1.5`의 "쌍 필수" 조건은 오탐 클릭은 줄이지만 실제 요구사항과 맞지 않는다.

보상상자는 다음 상태가 모두 가능하다.

- 상자가 하나도 없음
- 파란상자만 하나 있음
- 갈색상자만 하나 있음
- 파란상자와 갈색상자가 둘 다 있음

따라서 첫 탐색에서 반드시 쌍을 요구하면 단일 파란상자 또는 단일 갈색상자를 놓칠 수 있다.

## 현재 로컬 상태

`v0.1.5` 배포 후 로컬에서 추가 실험 변경을 시작했지만 아직 완성되지 않았다.

현재 미완성 변경 내용:

- `tbh_auto/constants.py`
  - 보상상자 픽셀 서명 검증용 설정값을 추가했다.
- `tbh_auto/rewards.py`
  - `reward_box_pair_required_without_row` 조건 사용을 제거했다.
  - 후보 crop의 흰색/갈색/파란색 픽셀 비율을 검사하는 보조 검증을 추가했다.

현재 실험 결과:

- 사용자가 제공한 "상자 없음" 이미지에서 전체 후보 탐색은 `None`으로 떨어지는 방향까지 확인했다.
- 하지만 파란상자만 찾는 순서에서는 아직 오탐이 남아 있다.
  - 사용자 이미지: 영웅창 아이콘 조각이 `reward_blue_box_bubble`처럼 잡힐 수 있음.
  - `logs/qa_after_ensure_cube_open.png`: 전투 하단 UI/이펙트 조각이 파란상자처럼 잡힐 수 있음.
- `logs/after_click_reward_chest.png`의 실제 단일 파란상자 화면에서도 형태 보정이 실제 상자보다 전투 이펙트를 먼저 고르는 문제가 있었다.
- 즉, 단순 색 비율 검증만으로는 충분하지 않다.

이 로컬 변경은 아직 커밋하거나 릴리즈하지 않는 것이 맞다.

## 이어서 할 작업

1. `v0.1.5`의 쌍 필수 방식은 최종 해결책으로 쓰지 않는다.
2. 보상상자 탐지는 다음 네 상태를 모두 지원해야 한다.
   - 없음
   - 파란상자 1개
   - 갈색상자 1개
   - 파란상자 + 갈색상자 2개
3. 형태 기반 fallback을 단순 색 비율이 아니라 "보상 말풍선 구조" 기준으로 다시 설계한다.
   - 후보 crop을 64x46 기준으로 정규화한다.
   - 실제 템플릿 alpha mask 영역과 구조 유사도를 계산한다.
   - 쌍이 있으면 pair bonus를 주되, 쌍이 없어도 충분히 확실한 단일 상자는 허용한다.
   - 영웅창 장비 아이콘, 삭제 말풍선, 큐브 UI, 전투 이펙트, 하단 파란 구슬 UI는 배제한다.
4. 후보를 하나만 즉시 고르지 말고 여러 후보를 수집한 뒤 점수화해서 가장 그럴듯한 후보를 선택한다.
5. 회귀 확인 샘플을 최소한 아래 이미지들로 돌린다.
   - `C:/Users/kkhan/AppData/Local/Temp/codex-clipboard-21b83c1d-6d83-4712-80a3-72ec4d2c5650.png`
   - `logs/qa_after_ensure_cube_open.png`
   - `logs/after_click_reward_chest.png`
   - `logs/qa_1x_box_loop_before.png`
   - `logs/qa_step_31_layout_bottom_scan.png`
   - `logs/qa_step_36_layout_right_bottom_scan.png`
6. 통과 기준을 정한다.
   - 상자 없음 화면: `None`
   - 단일 파란상자 화면: 실제 파란상자 좌표
   - 단일 갈색상자 화면: 실제 갈색상자 좌표
   - 두 상자 화면: 둘 중 하나를 실제 좌표로 감지하고, 다음 클릭에서 남은 상자를 감지
7. 검증 후 `python -m compileall tbh_auto`, `git diff --check`, `cmd /c build_exe.bat`를 돌린다.
8. 문제가 해결되면 새 커밋을 만들고 `v0.1.6` 릴리즈로 올린다.

## 주의

- 현재 GitHub 최신 릴리즈는 `v0.1.5`이다.
- `v0.1.5`는 "상자 없음" 오탐 일부를 막지만 단일 상자 케이스를 놓칠 수 있으므로 다음 작업에서 반드시 고쳐야 한다.
- 오늘 추가한 로컬 실험 변경은 아직 최종안이 아니다.

## 이어서 진행한 작업

- `v0.1.5`의 쌍 필수 조건은 제거했다.
- 보상상자 fallback 탐지를 다음 방식으로 다시 설계했다.
  - 흰색 말풍선 후보뿐 아니라 파란 보상상자의 밝은 파란 배경도 후보 마스크에 포함했다.
  - 후보 crop을 실제 보상상자 템플릿과 비교하는 구조 유사도 점수를 추가했다.
  - 색 비율 점수는 1차 후보 필터로만 쓰고, 최종 통과는 템플릿 구조 점수로 판단한다.
  - 쌍이 있으면 pair bonus는 유지하지만, 쌍이 없어도 충분히 확실한 단일 파란상자/갈색상자는 허용한다.
- 진단 스캔(`tbh_auto/diagnostics.py`)에서도 같은 템플릿 구조 검증을 쓰도록 연결했다.
